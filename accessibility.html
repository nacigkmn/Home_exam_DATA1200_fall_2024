Accessibility

Design

Accessibility was the most focused points while we were creating our project. Principles of universal design was our guide lines to create a most possible user friendly web design. Therefore we tried to apply seven design principles (Equitable use, flexibility use, simple and intuitive use, perceptible information, tolerance for error, low physical effort and appropriate size and space for approach and use). We used variety of sources to get deeper knowledge about these principles and one of them was reachable here.

Images

All images are downloaded from free image providing websites. Links are stated below images and visible on website.

Code structure

We tried to write understandable and clear code through the project. Project has least possible files and all of them easy to read. At the beginning we create one css file for each html files. In addition to that we thought it could better to create separate @media css and layout css which includes shared codes for each page (such as menu bar, body background, text plains and footer.) But this increased amount of files in four project more than double. So we decided to create one css file where all sections are separated with comment lines. We believe that all codes in css file is neat and easy to find. For easier navigation in CSS document we decided to place common parts like body, header and footer at the top of the CSS code. Table of content is also usefull to find out each parts of code.

Testing

We tested our project by some discussions with people who has more experience about web development. We gathered critics and information and tried to apply them to our project if its possible. If it was not possible to apply idea direct we focused on how to adapt idea to our project or how can we adapt our project to idea.

Finally we ran google lighthouse test. Here in the images the test results of our index page for both desktop and mobile are visible. Rest of the pages were also tested and many fails were fixed.

Conclusion

Our web project is user friendly and simple. Esthetic sides can be relative and criticized. Code is neat, easy to understand and modified for to use it possible feature projects.
